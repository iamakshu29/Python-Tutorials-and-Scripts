A todo app with users
    register/login
    each user has their own todos
    full CRUD on todos
    only logged in users can access their own todos

DATA MODEL
    A Todo entity with fields like id, title, description, status (pending/in-progress/done), priority, due_date, created_at, updated_at, and optionally user_id for multi-user support.

REST API Endpoints - DONE
    GET /todos — fetch all todos (with filtering, sorting, pagination)
    POST /todos — create a new todo
    GET /todos/:id — fetch a single todo
    PUT/PATCH /todos/:id — update a todo (title, status, etc.)
    DELETE /todos/:id — delete a todo
    PATCH /todos/:id/complete — mark as complete (optional dedicated endpoint)
